## Problem Identified

The featured meeting API (`https://mnybvagd5m.execute-api.eu-west-1.amazonaws.com`) is returning picks that **don't exist in the main SureBetBets DynamoDB table**.

This means one of three things:

### Option 1: Separate Featured Picks System
- Featured meetings use a different picking algorithm
- Data stored in separate DynamoDB table or attribute
- API queries different source than main picks

### Option 2: Data Selection Issue
- Featured picks ARE in SureBetBets but with different attributes
- API filters by `is_featured=true` or similar flag
- We're querying wrong attributes

### Option 3: Manual Curation
- Featured picks are manually selected
- Not automatically generated by system
- Stored separately from main algorithm picks

---

## Most Likely Scenario

Based on the evidence, **Option 2 is most likely**:

The picks probably exist in DynamoDB with a flag like:
- `is_featured_meeting=true`
- `meeting_type='featured'`
- `selection_policy='featured_meeting'`

When I queried earlier, I found:
- Gloriously Glam with `bet_id: 2026-05-20_GOWRAN_17:10_56134859`
- Lady Mairen at 18:20 (not 19:20) with `selection_policy: 'model_top_score'`
