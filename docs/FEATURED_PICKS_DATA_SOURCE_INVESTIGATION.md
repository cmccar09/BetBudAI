## Most Likely Explanation

### Featured Meeting Lambda Architecture

```
User Request
    ↓
API Gateway (/api/picks/featured-meeting)
    ↓
Lambda: surebet-featured-meeting
    ↓
    ├─→ Query SureBetBets for course picks
    ├─→ Run featured algorithm on course
    ├─→ Apply expert curation logic
    ├─→ Format as featured picks
    ↓
Return JSON (not stored in DB)
```

**Why This Makes Sense**:
1. Explains why picks don't exist in database before API call
2. Allows dynamic featured course selection
3. Enables expert logic layer on top of algorithm
4. Results stored in cache/memory, not persisted

**Why This Is Problematic**:
1. No audit trail of what was actually recommended
2. Can't backtest featured performance
3. Results might change between API calls
4. Learning system can't analyze featured picks

## Recommended Solution

### Short-Term Fix (Immediate)

**Persist featured picks to database when served**:

```python
# In surebet-featured-meeting Lambda
def handler(event, context):
    # Generate featured picks
    picks = generate_featured_picks(course, date)

    # Store to DynamoDB for audit trail
    for pick in picks:
        table.put_item(Item={
            'bet_date': date,
            'bet_id': f'{date}_FEATURED_{course}_{pick.race_time}_{pick.horse}',
            'is_featured_meeting': True,
            'selection_policy': 'featured_meeting',
            **pick.to_dict()
        })

    # Return picks
    return {
        'statusCode': 200,
        'body': json.dumps(picks)
    }
```

**Benefits**:
- Creates audit trail
- Enables backtest analysis
- Allows learning system integration
- Maintains data integrity

### Long-Term Fix (Next Sprint)

**Integrate featured picks into main pipeline**:

1. **Morning Pipeline** generates both main + featured picks
2. **Featured picks** stored with `is_featured_meeting=True`
3. **API** queries database (not on-demand generation)
4. **Results** settled same way as main picks
5. **Learning** processes featured picks separately

**Benefits**:
- Single source of truth
- Consistent data model
- Full historical tracking
- Proper performance attribution
