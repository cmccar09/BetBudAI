## The 7-Factor Comprehensive Analysis System

Located in: `comprehensive_pick_logic.py`

### Factors Analyzed (100 points total)

1. **ODDS_SCORE** (20 pts max)
   - Sweet spot: 3.0 - 10.0 odds
   - Penalizes extreme long shots and short-priced favorites
   
2. **FORM_SCORE** (20 pts max)
   - Recent race performance
   - Win/place history
   - Consistency

3. **JOCKEY_SCORE** (15 pts max)
   - Jockey's recent win rate
   - Partnership with trainer
   - Course/distance expertise

4. **TRAINER_SCORE** (15 pts max)
   - Trainer's recent form
   - Strike rate at venue
   - Historical performance

5. **COURSE_DISTANCE_SCORE** (10 pts max)
   - Horse's record at this course
   - Performance at this distance
   - Going preference

6. **WEIGHT_SCORE** (10 pts max)
   - Actual weight vs optimal
   - Weight trend analysis

7. **PACE_SCORE** (10 pts max)
   - Early/late pace preference
   - Race shape suitability

### 4-Tier Grading System

After analysis, picks are graded:

| Grade | Points | Display | Stake Multiplier | UI Visibility |
|-------|--------|---------|------------------|---------------|
| EXCELLENT | 75+ | 🟢 Green | 2.0x | Always shown |
| GOOD | 60-74 | 🟠 Light amber | 1.5x | Always shown |
| FAIR | 45-59 | 🟤 Dark amber | 1.0x | Always shown |
| POOR | <45 | 🔴 Red | 0.5x | Hidden from UI |

**Mandatory Coverage Rule**: System must analyze ≥90% of horses in a race, or the entire race is SKIPPED.

## Troubleshooting Guide

### Problem: No picks showing on website

**Check 1**: Did workflow run today?
```bash
aws dynamodb get-item --table-name SureBetBets --key '{"bet_date":{"S":"2026-05-21"},"bet_id":{"S":"WORKFLOW_RUN_LOCK"}}' --region eu-west-1
```

If `status` = "completed", workflow ran successfully.

**Check 2**: Are picks in DynamoDB?
```bash
aws dynamodb query --table-name SureBetBets --key-condition-expression "bet_date = :date" --expression-attribute-values '{":date":{"S":"2026-05-21"}}' --region eu-west-1 --max-items 5
```

Should show pick records.

**Check 3**: Is API Lambda working?
```bash
curl https://[your-api-gateway-url]/api/picks/today
```

Should return JSON with picks.

**Check 4**: Frontend cache
- Hard refresh: Ctrl+F5
- Check browser console for errors

### Problem: Workflow fails with coverage error

```
⚠️ Race skipped: Only 65% coverage (minimum 90% required)
```

**Cause**: Betfair API didn't return full runner list, or data fetch failed.

**Fix**: 
1. Check internet connection
2. Verify Betfair API credentials
3. Re-run workflow - often resolves on retry

### Problem: "WORKFLOW_RUN_LOCK already exists" error

**Cause**: Lock from previous run wasn't released (script crashed).

**Fix**:
```bash
aws dynamodb delete-item --table-name SureBetBets --key '{"bet_date":{"S":"2026-05-21"},"bet_id":{"S":"WORKFLOW_RUN_LOCK"}}' --region eu-west-1
```

Then re-run workflow.

## Performance Optimization Notes

### What Makes a Good Pick?

From analyzing the 57.8% ROI performance:

1. **Sweet Spot Odds**: 4.0 - 9.0 (moderate underdogs)
2. **Comprehensive Score**: 70+ points
3. **Multiple Strong Factors**: At least 3 factors scoring 12+ points
4. **Avoid**:
   - Favorites <3.0 odds (low value)
   - Long shots >15.0 odds (unreliable)
   - Horses with <10 points jockey+trainer combined

### Learning System

The `daily_learning_cycle.py` updates ML weights based on:
- Yesterday's wins/losses
- Jockey/trainer strike rate trends
- Venue-specific form patterns
- Seasonal adjustments

This is why yesterday's results must be fetched before generating today's picks.
